# Testing Requirements

## Общие требования
- Каждый компонент должен иметь хотя бы один unit-тест.
- Покрытие кода тестами — не менее 80% для новых компонентов.
- Тесты должны быть детерминированными (без случайных данных).

## Unit-тесты
- Используйте **Vitest** или **Jest**.
- Тестовый файл должен находиться рядом с компонентом: `Button.test.tsx`.
- Имя теста должно описывать проверяемое поведение: `it('should call onClick handler when clicked')`.

## Структура теста
- Используйте паттерн **Arrange-Act-Assert** (Подготовка — Действие — Проверка).
- Один тест — одна проверка.
- Не используйте условную логику внутри тестов (`if`, `for`).

## Тестирование компонентов
- Используйте **React Testing Library** для тестирования компонентов.
- Проверяйте поведение, а не реализацию.
- Не тестируйте внутреннее состояние компонента напрямую.

## Тестирование обработчиков
- Обработчики событий должны быть протестированы через fireEvent или userEvent.
- Проверяйте, что обработчик вызван с правильными аргументами.

## Интеграционные тесты
- Ключевые пользовательские сценарии должны быть покрыты интеграционными тестами.
- Интеграционные тесты должны проходить в CI/CD пайплайне.

## Моки
- Внешние зависимости должны быть замоканы.
- Не мокайте то, чем не владеете (предпочитайте faker-данные).
- Моки должны быть реалистичными.
